## 正文

秋招投了快50份简历，面试一只手数得过来。

复盘的时候发现一个大问题——我用的是同一份通用简历，意向写的是"产品运营"，海投了数据分析、AI运营、供应链财务、电商运营……各种方向的岗位。

HR看到的是一份跟JD关键词几乎没交集的简历，ATS机器筛直接pass了。

**后来我干了件事：**

我自己动手做了一个Claude Code技能（免费的），每次投一个新岗位，把JD（岗位描述）和我的简历丢进去，它帮我做三件事：

📍 **拆解JD** —— 这个岗位到底要什么人，硬性要求是什么，关键词有哪些，隐藏的需求是什么

📍 **逐条匹配** —— 我的每段经历跟JD的每条要求一一对标，哪些是强匹配、哪些是部分匹配、哪些是差距，清清楚楚

📍 **直接在原始简历上改文字** —— 排版、字体、表格完全不动，只改表述。把"信贷档案审查"改成"企业数据交叉核验"，把"用Python处理数据"改成"利用AI工具推动数据处理自动化"——同一段真实经历，换了个JD要的说法

**核心原则：绝不编造。** 不会给你加没做过的项目、没用过的工具。只在你有的事实基础上，用JD的语言重新讲一遍。

---

**效果有多明显？**

我拿自己简历测了两个岗位：

| 岗位 | 修改前匹配度 | 修改后匹配度 |
|------|------------|------------|
| 存货财务分析岗 | ~30%（意向写的还是产品运营） | ~50% |
| AI内容运营岗 | ~40% | ~80% 🎯 |

第二个岗改完之后，实习经历里"ChatGPT辅助编写Python脚本实现数据批处理"这句话直接命中了JD里"会用AI工具、愿意深耕"的要求——这段经历简历上一直有，但原来写得太低调了。

---

**怎么用（免费，开源）：**

你电脑上装好 Claude Code 之后，终端敲一行：

```
git clone https://github.com/KE-1027/resume-tailor-skill.git ~/.claude/skills/resume-tailor-skill
```

然后打开 Claude Code，输入：

```
/resume-tailor-skill
```

把你要投的JD粘贴进去，再发你的简历文件路径，等两三分钟，定制好的DOCX直接生成，排版和原始简历一模一样。

---

**适合谁：**

- 海投简历回复率低、想针对每个岗位定制的
- 跨方向投递（一段实习要写成好几种版本）
- 不知道怎么把经历"翻译"成JD语言的
- 不想花钱找简历修改服务的

---

**最后说句实话：**

这个东西不是魔法。如果你的经历跟岗位真的一点关系没有，它不会凭空给你编。但它能帮你在诚实的前提下，把你的真实经历用最匹配JD的方式讲出来——而这件事，恰恰是大部分海投简历没做到的。
